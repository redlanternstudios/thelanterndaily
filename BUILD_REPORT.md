## Primary Blocker

**Next.js 16 Turbopack framework bug:** Turbopack's tree-shaker strips React runtime context when statically generating the special `_global-error` route boundary. This manifests as:

```
TypeError: Cannot read properties of null (reading 'useContext')
```

Confirmed across multiple remediation attempts:
- Minimal file content (no imports beyond `'use client'`)
- Added `export const dynamic = 'force-dynamic'`
- Edge runtime config
- Removed file entirely (Next.js auto-injects default)
- Upgraded to `next@16.3.0-preview.3`
- Set `TURBOPACK=0`, `NEXT_IGNORE_TURBOPACK=1`, `NODE_ENV=production`

None of these bypassed the Turbopack crash.

## Secondary Blocker (Next.js 15 downgrade)

Downgrading to `next@15.2.4` bypasses Turbopack but surfaces:
1. **ESLint dependency:** `es-abstract/2024/AddEntriesFromIterable` missing (transitive dep of `eslint-plugin-react` via `eslint-config-next`)
2. **Pages Router conflict:** `_error` handler imports `HtmlContext` from `next/document`, causing: `Error: <Html> should not be imported outside of pages/_document` — crashes when prerendering `/404`
